## 組裝步驟（手冊精簡版）

### A. 機構（Step 1–9）

1. **撕掉所有壓克力件的保護膜**（霧面那層，含底盤與 T 型小片）— 新手最常忘
2. 底盤「**MAKER**」字樣那面朝上
3. 取 4 片 **T 型固定片**（也撕膜）
4. 備好馬達 + 圓頭螺絲/螺帽
5. T 型片裝到底盤**兩側**
6. 馬達放兩片 T 中間，**對孔鎖上**（先不鎖死，兩邊都上再對齊鎖緊）
7. 另一邊同樣；**馬達線朝車尾方向**
8. **輪子**壓進馬達軸（D 形孔，對準缺口壓到底）
9. **馬眼輪**（球輪）用 2 根長銅柱 + 4 螺絲鎖在**車尾底面**

### B. 電子件上座（Step 10–17）

10–13. **Uno** 用 4 根短銅柱鎖上（右下角那孔不鎖，因下方有馬達）
14–16. **L298N** 用 4 根短銅柱 + 8 螺絲鎖上
17. **電池盒**裝上（與 L298N 之間留 ~3mm 間距）

### C. 接線（Step 18–20）

18. 電池盒 **DC 頭 → 插進 Uno 的 DC 座**
19. **馬達線 + 電池紅黑線 → 接到 L298N**
20. 完成 🎉

## 接線腳位（接線時對照）

> ⚠️ **動手前先拔掉 L298N 的 ENA、ENB 兩個跳線帽（jumper）**，否則杜邦線插不上、無法 PWM 控速。其他跳線帽不動。

**信號線 6 條（L298N → Arduino，插擴展板對應腳的 S 排）：**

| L298N 腳 | Arduino | 程式名 |
|---|---|---|
| IN1 | D8 | LEFT1 |
| IN2 | D9 | LEFT2 |
| ENA | D10 | LEFT_PWM（左輪速度） |
| IN3 | D7 | RIGHT1 |
| IN4 | D6 | RIGHT2 |
| ENB | D5 | RIGHT_PWM（右輪速度） |

> 記法：左輪 8/9/10、右輪 7/6/5。

**電源 3 條（6 節約 9V / 18650 約 11V）：**

| 從 | 到 |
|---|---|
| 電池盒紅線 | L298N +12V |
| 電池盒黑線 | L298N GND |
| 電池盒 DC 圓頭 | Arduino DC 圓孔 |

> 這樣接地自動共用（電池負極同接兩邊），不用另拉地線。
> （若是 4 節約 6V 電池盒：另需 L298N 5V→Arduino 5V、GND→GND。）

**馬達 2 顆：** 左馬達 → OUT1/OUT2；右馬達 → OUT3/OUT4。某輪轉反向就把該馬達兩線對調。

> 馬達銅片很脆：線是焊好再用熱熔膠固定的（應力釋放），接線時別拉扯馬達線。

## 模組要全裝嗎？裝到什麼程度（以割草機目標）

| 模組 | 教學 | 專案需要? | 說明 |
|---|---|---|---|
| 底盤+馬達+輪+L298N+電池 | arduino-car-l298n | ✅ 必裝（核心） | 會動的車，一切基礎 |
| 超音波 SR04 | arduino-car-sr04 | ✅ 保留，可延後 | 近距離避障，Arduino 的即時反射層。可等 Pi 跑通後再補裝 |
| 藍牙 HC-05 | arduino-car-bt | ⛔ 可跳過 | 原為手機遙控/測試；但 USB 序列埠能測一樣的馬達協定，藍牙等於多繞一圈 |
| 紅外循跡 | arduino-car-ir | ⛔ 可跳過 | 「沿黑線走」；本車靠視覺自由覆蓋整片，用不到 |

- **專案主線**：馬達(L298N) → 接 Pi+YOLO 自走 → 之後補超音波當安全反射層。
- **循跡為何用不到**：循跡＝跟著地上黑線跑（像軌道）；割草＝自己決定走遍整片（覆蓋演算法 + 視覺），不同思路。

### 決策紀錄（2026-06-10）

- **藍牙 HC-05 → 砍掉**：它唯一的價值是「測試馬達指令協定（w/a/s/d/x）」，但這件事用 **USB 序列埠**就能做（Uno 接筆電、開序列埠監控打字母，測的是跟之後 Pi 一模一樣的 `Serial.read()` code path）。USB 線本來就要接，藍牙等於多繞一圈，還省掉 HC-05 的 RX 5V 分壓坑。唯一損失：無法用手機無線手動接管（目前不需要）。
- **超音波 SR04 → 保留但可延後**：它跟視覺**不是重複**，是不同層——超音波是毫秒級、不受光線影響的即時反射（「前面 20cm 有東西，停！」），視覺是 5fps + 延遲的聰明判斷（這是草/人/花圃，往哪繞）。對應 [hardware-roadmap.md](hardware-roadmap.md) 兩個腦分工：Arduino 顧反射、Pi 顧導航。少了它 Arduino 會退化成啞巴馬達板。成本極低（一顆幾十元 + 4 條線），且階段 4 裝刀片時的安全架構跑不掉。策略：先讓 Pi 開車跑通，再把超音波補上（獨立模組，隨時加）。

## Pi 與 Arduino 怎麼溝通（重點）

兩者**都裝在同一台車上（相距約 10cm）**，所以：

- ✅ **Pi ↔ Arduino → 用 USB 線（序列埠）**：簡單、最穩、插上就通。無線是「遠距離」才有價值，疊在一起拉藍牙沒意義還要忍配對麻煩。
- ✅ **藍牙 HC-05 的角色 → 人用手機/筆電「隔空遙控」車**（階段 1 測試 / 手動接管），不是 Pi↔Arduino。
- 程式幾乎一樣：藍牙 `BT.read()`、USB `Serial.read()`，馬達邏輯不變。
- 詳見 [hardware-roadmap.md](hardware-roadmap.md) 的「Pi ↔ Uno 怎麼合體」。

## 開工順序（對應路線圖階段 1）

1. 組機構（A）→ 車能三點著地站起來
2. 上電子件（B）
3. 接線（C）
4. 上傳官方 **L298N 範例** → 車會前進/後退/左/右/停（驗證硬體全通）

5. 之後接 Pi + webcam 跑 YOLO（超音波避障延後、藍牙跳過），見 [hardware-roadmap.md](hardware-roadmap.md) 的「Pi ↔ Uno 怎麼合體」。
