# 實體機器路線圖（硬體 / 韌體）

> 狀態：**方向已定，尚未動工**
> 建立日期：2026-06-03
> 延伸自 [vision-yolo.md](vision-yolo.md) 的討論

## 重大決定：要做「真的會走的機器」

GreenKeeper 不只停在模擬器，目標是做出**一台真實、會自己走的車**（割草刀片是最後最後的事，前期先不裝）。

模擬器不會白做——它是這台機器的「**大腦練習場**」。

## 三層架構

一台自走機器拆成三層：

| 層 | 比喻 | 做什麼 | 例子 |
|---|---|---|---|
| ① 感知 | 眼睛 | 看前面有什麼 | 鏡頭拍照 → YOLO 判斷「草 / 障礙物」 |
| ② 決策 | 大腦 | 決定怎麼走 | 「左邊還沒割，往左轉」 |
| ③ 動作 | 手腳 | 把決定變成動作 | 「左輪慢、右輪快」→ 車子左轉 |

## 模擬器目前的定位

模擬器**只做了第 ② 層（大腦），而且是「上帝視角作弊版」**：

- **沒有真的輪子**：每一格時間把圓點「瞬移」一小段（`sim.js` 的 `this.x += cos(angle)*speed`），沒有馬達 / 轉速。
- **沒有真的眼睛**：演算法直接讀整張地圖陣列，等於早就知道答案，所以不需要 YOLO。

真機要做的事 = 把大腦左右兩端的「假器官」換成真的：

```
        ┌─────────── 真機 ───────────┐
眼睛 → │  鏡頭+YOLO  →  大腦  →  馬達  │ → 輪子
        └────────────────────────────┘
模擬器 │   (假sensor)   大腦  (假瞬移)  │
        └────────────────────────────┘
            ↑換成真的          ↑換成真的
```

中間的大腦在模擬器裡先調好（這也是 vision-yolo.md 的路線 B），真機只是換掉兩端。

## 真機需要的東西

### 🦴 身體（會走的車）
- 底盤 + 兩個輪子各一顆馬達 → 兩輪轉速不同就會轉彎（差速轉向，坦克原理）
- 馬達驅動板（motor driver）→ 馬達吃電大，主機板小電流推不動，需要它當中間放大器
- 電池

### 👁 眼睛
- 一顆 USB 鏡頭（視訊鏡頭等級即可）

### 🧠 兩個腦（新手最容易卡的觀念）
真機通常**兩個處理器分工**：

| | 誰 | 負責 | 特性 |
|---|---|---|---|
| 大腦 | 樹莓派 / Jetson | 跑 YOLO、做決策 | 算力大但反應慢 |
| 小腦 | Arduino / ESP32 | 即時控制輪速、讀避障感測器 | 反應快但笨 |

**為什麼分兩個**：看影像耗算力會卡頓，但控制輪子必須毫秒級即時不能等。所以讓快但笨的小腦顧輪子，慢但聰明的大腦看路，兩者用一條序列線溝通。

> ⚠️ 割草刀片很危險。第一階段只做「會自己走的車」，**不裝刀片**。先解決導航。

## 分階段路線圖（每階段都有可動成果）

1. **遙控車** — 車能動、用鍵盤/手機能開。先搞定輪子、馬達、馬達驅動板。（純硬體，還沒 AI）
2. **眼睛接上** — 鏡頭畫面進電腦，YOLO 能框出「草 / 障礙物」。（單獨玩 YOLO，先不上車）
3. **接大腦** — 把模擬器調好的決策演算法，改成吃「YOLO 看到的東西」，自動開車。
4. **（很久以後）** 裝刀片、防水、電量 / 回充座。

## Pi ↔ Uno 怎麼合體（階段 3）

三層各就各位，這台車（Uno 套件）= 手腳，Pi + webcam = 眼睛 + 大腦：

```
👁 webcam ──► 🧠 Pi 4 ──(指令)──► 🦿 Uno + L298N ──► 輪子
  (眼睛)      (大腦,跑YOLO)        (自走車套件 = 手腳)
```

**怎麼溝通：一條 USB 線走「序列埠 (Serial)」**（最簡單、最穩、重用套件的 USB 線）：

- Pi 的 USB ↔ Uno 的 USB 接起來，就像一根「傳紙條的管子」。約定 baud rate（如 9600）。
- Pi（Python）寫指令：`ser.write(b'a')` → Uno（`Serial.read()`）讀到 `'a'` → 左轉。
- **跟套件的藍牙範例幾乎一樣**，只是把 `BT.read()` 換成 `Serial.read()`，馬達邏輯不動。
- 雙向：Uno 也能把超音波距離寫回管子給 Pi 參考。

指令對照（沿用範例）：`w` 前進／`x` 後退／`a` 左轉／`d` 右轉／`s` 停。

> 其他方式（先別用）：GPIO TX/RX 直連需處理 **3.3V(Pi) vs 5V(Uno) 電壓轉換**，否則燒 Pi；藍牙可無線但 Pi 配對較麻煩。**USB 序列埠最省事。**

⚠️ **供電分開**：USB 線只走資料。**電池盒推 Uno + 馬達；Pi 用自己的行動電源。** 否則馬達一啟動電壓掉，Pi 會當機重開。

## 心理準備
- 時間：好幾個月。
- 花費：約台幣 3,000～10,000。
- 動手：需要接線（可能要一點焊接）。
- 策略：分階段做，每階段都有會動的成果，避免卡死。
